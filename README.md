# GenAI-GDELT

Framing analysis of generative AI governance in online news, using the GDELT 2.0 Global Knowledge Graph (2022–2026).

We examine how news media framed the governance of generative AI across ~1.1 million articles spanning November 2022 (ChatGPT launch) through June 2026, using a combination of dictionary-based frame detection and ML embedding scoring.

**Research questions**
- **RQ1** How did coverage volume evolve over time?
- **RQ2** Which governance frames dominated, and how did their share shift?
- **RQ3** Did major policy milestones (EU AI Act, Bletchley Summit, …) trigger detectable frame shifts?

## Quickstart — reproduce figures from pre-computed data

This path requires no BigQuery account. Download the pre-processed data from Google Drive and run the figure notebook.

**1. Install dependencies**

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

**2. Download the large data files**

```bash
python scripts/download_data.py
```

This places three files in the repo:

| File | Size | Contents |
|------|------|----------|
| `data/raw/gdelt_genai_gov.csv` | ~4.7 GB | Raw GDELT corpus (1.1 M articles) |
| `data/raw/monthly_genai_total.csv` | ~4 KB | Monthly article totals (normalisation baseline) |
| `data/processed/ml_frame_scores_embedding_full.parquet` | ~117 MB | ML frame scores + embeddings |

**3. Regenerate the aggregated parquets**

```bash
jupyter nbconvert --to notebook --execute notebooks/01_pipeline.ipynb
```

Outputs land in `data/processed/` (monthly\_frames.parquet, event\_studies.parquet, etc.).

**4. Produce the paper figures**

```bash
jupyter nbconvert --to notebook --execute notebooks/03_paper_figures.ipynb
```

Figures are written to `paper/figures/`.

## Full replication — re-extract from BigQuery

Use this path to reproduce everything from the raw GDELT source.

**Prerequisites:** a Google Cloud project with the BigQuery Sandbox enabled (free, no credit card). See `docs/credentials_setup.md` for step-by-step instructions.

**1. Authenticate**

```bash
gcloud auth login
gcloud config set project YOUR_PROJECT_ID
gcloud auth application-default login
```

Copy `.env.example` to `.env` and set:
```
BIGQUERY_PROJECT=your-project-id
```

**2. Run the extraction query**

Open `queries/extract_genai_gov.sql` in the BigQuery console (or run via `bq` CLI). Expected scan: ~500 GB–1 TB — within the free tier monthly limit, but check the estimate in the console before running.

Save the result as `data/raw/gdelt_genai_gov.csv`.

**3. Preprocess the raw CSV**

```bash
python scripts/preprocess_raw.py
```

Streams the 4.7 GB CSV in 100 K-row chunks, applies frame detection, and writes `data/interim/gdelt_preprocessed.parquet` (~2 GB).

**4. Run the full notebook pipeline**

```bash
jupyter nbconvert --to notebook --execute notebooks/01_pipeline.ipynb
jupyter nbconvert --to notebook --execute notebooks/02_ml_framing.ipynb
jupyter nbconvert --to notebook --execute notebooks/03_paper_figures.ipynb
```

`02_ml_framing.ipynb` requires the optional ML dependencies (see below) and a GPU is recommended but not required.

The embedding-based frame scoring (`notebooks/02_ml_framing.ipynb`, `src/framing_scores.py`) uses `sentence-transformers`. Install the extras after installing the base requirements:

```bash

pip install torch --index-url https://download.pytorch.org/whl/cpu

# NVIDIA GPU (CUDA 12.1)
```

## Frame taxonomy

Six governance frames are detected via keyword matching in `data/lexicons/frames.yaml` and validated by ML scoring:

| Frame | Focus |
|-------|-------|
| `innovation_opportunity` | Progress, benefits, economic growth |
| `risk_safety` | Harms, threats, existential concerns |
| `regulation_governance` | Laws, oversight, compliance frameworks |
| `rights_privacy` | Civil rights, data protection, surveillance |
| `economic_competition_labour` | Jobs, market concentration, automation |
| `misinformation_integrity` | Deepfakes, false content, trust erosion |

Keyword dictionaries are multilingual (EN, FR, DE, ES, IT, NL, PL, PT). Edit `data/lexicons/*.yaml` to change frame definitions; run `python -m src.build_query --write` afterwards to keep the SQL in sync.

## Corpus definition

- **Source:** GDELT 2.0 GKG (`gdelt-bq.gdeltv2.gkg_partitioned`)
- **Date range:** 2022-11-01 – 2026-06-30
- **Filter:** article must mention at least one generative-AI term (from `data/lexicons/genai.yaml`) AND at least one governance signal (keyword from `data/lexicons/governance.yaml`, GDELT theme tag, or URL slug)
- **Size:** ~1.1 M articles
